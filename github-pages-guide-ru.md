# Руководство: публикация проекта через GitHub Pages

## Обзор

**GitHub Pages** — это сервис, который позволяет размещать статические сайты напрямую из GitHub-репозитория.

Он подходит для:

* документации
* портфолио
* лендингов
* учебных проектов

Сайт автоматически обновляется при изменении репозитория.

## Быстрый старт

### Минимальный набор команд

```bash id="gq1p9a"
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

---

### Включение GitHub Pages

1. Откройте репозиторий
2. Перейдите в **Settings**
3. Откройте раздел **Pages**
4. Выберите:

   * Source: `Deploy from branch`
   * Branch: `main`
   * Folder: `/ (root)`
5. Нажмите **Save**

---

## Требования

Перед началом убедитесь, что у вас есть:

* аккаунт GitHub
* установленный Git
* проект с файлом `index.html`

> ⚠️ GitHub Pages работает только со статическими сайтами (HTML/CSS/JS)

---

## Создание репозитория

1. Перейдите на GitHub
2. Нажмите **New repository**
3. Укажите:

   * Repository name
   * Visibility (public или private)

Дополнительные опции:

* README (опционально)
* .gitignore (опционально)
* License (опционально)

## Связывание с GitHub

Добавьте удалённый репозиторий:

```bash id="v3kq1d"
git remote add origin https://github.com/USERNAME/REPO.git
```

Отправьте изменения:

```bash id="m8xq2z"
git push -u origin main
```

---

## Публикация через GitHub Pages

После пуша:

1. Откройте Settings репозитория
2. Перейдите в Pages
3. Выберите ветку `main`
4. Сохраните настройки

Через 1–5 минут сайт будет доступен по адресу:

```
https://USERNAME.github.io/REPOSITORY_NAME/
```

## Структура проекта

Минимальная структура:

```text id="xq8m1p"
project/
│
├── index.html
├── style.css
├── script.js
└── README.md
```

> ⚠️ Файл `index.html` должен находиться в корне проекта

## Частые проблемы

### 1. Сайт не открывается

Проверьте:

* включён ли GitHub Pages
* выбран ли branch `main`
* существует ли `index.html`

---

### 2. Ошибка "not a git repository"

```text
fatal: not a git repository
```

Решение:

```bash
git init
```

---

### 3. Изменения не отображаются

Решение:

* подождать 1–5 минут
* обновить страницу
* проверить `git push`

---

### 4. Неправильная ссылка

Убедитесь, что:

* username совпадает с GitHub аккаунтом
* имя репозитория указано верно

---

## Рекомендации

### 1. Используйте понятные имена коммитов

```bash
git commit -m "add landing page layout"
```

---

### 2. Проверяйте структуру перед публикацией

* index.html в корне
* нет лишних вложенных папок

---

### 3. Делайте отдельную ветку для разработки (опционально)

```bash
git checkout -b dev
```
