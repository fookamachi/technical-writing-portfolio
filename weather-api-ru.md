# Weather API (OpenWeatherMap) — Документация

## Обзор

**Weather API** предоставляет доступ к погодным данным через HTTP-запросы. Сервис позволяет получать текущую погоду, прогнозы и дополнительные метеоданные для использования в веб-приложениях, мобильных приложениях и сервисах.

Провайдер: **OpenWeatherMap**
Формат ответа: **JSON**
Протокол: **HTTPS**

## Быстрый старт

### Получение текущей погоды

```bash
curl "https://api.openweathermap.org/data/2.5/weather?q=London&appid=YOUR_API_KEY&units=metric"
```

### Пример ответа

```json
{
  "weather": [
    {
      "description": "light rain"
    }
  ],
  "main": {
    "temp": 18.5,
    "humidity": 82
  },
  "name": "London"
}
```

## Аутентификация

Для работы с API требуется API ключ.

**API Key** — уникальный идентификатор, используемый для авторизации запросов.

### Как получить ключ:

1. Зарегистрируйтесь на сайте OpenWeatherMap
2. Перейдите в раздел API Keys
3. Сгенерируйте ключ

Передаётся через параметр:

```
appid=YOUR_API_KEY
```

## Эндпоинты

### Текущая погода

```
GET /weather
```

Получение текущих погодных условий.

---

### Прогноз (5 дней)

```
GET /forecast
```

Возвращает прогноз погоды с интервалами.

---

## Параметры запросов

Запрос должен содержать хотя бы один параметр местоположения.

### Основные параметры

| Параметр | Тип    | Обязательный | Описание                     |
| -------- | ------ | ------------ | ---------------------------- |
| q        | string | нет*         | Название города              |
| lat      | float  | нет*         | Широта                       |
| lon      | float  | нет*         | Долгота                      |
| zip      | string | нет*         | Почтовый индекс              |
| id       | int    | нет*         | ID города                    |
| appid    | string | да           | API ключ                     |
| units    | string | нет          | metric / imperial / standard |
| lang     | string | нет          | язык ответа                  |

> *Обязательно указать хотя бы один параметр локации (q, lat/lon, zip или id)

---

## Примеры запросов

### По названию города

```bash
curl "https://api.openweathermap.org/data/2.5/weather?q=Amsterdam&appid=YOUR_API_KEY&units=metric"
```

### По координатам

```bash
curl "https://api.openweathermap.org/data/2.5/weather?lat=52.3676&lon=4.9041&appid=YOUR_API_KEY"
```

## Формат ответа

### Основные поля

| Поле                  | Тип    | Описание        |
| --------------------- | ------ | --------------- |
| main.temp             | number | Температура     |
| main.humidity         | number | Влажность       |
| weather[].description | string | Описание погоды |
| wind.speed            | number | Скорость ветра  |
| sys.sunrise           | number | Время восхода   |
| sys.sunset            | number | Время заката    |

## Ошибки

### Коды ответов

| Код | Значение            |
| --- | ------------------- |
| 200 | Успешный запрос     |
| 400 | Некорректный запрос |
| 401 | Неверный API ключ   |
| 404 | Город не найден     |
| 429 | Превышен лимит      |
| 500 | Ошибка сервера      |

---

### Пример ошибки

```json
{
  "cod": "404",
  "message": "city not found"
}
```

---

## Лимиты

### Бесплатный тариф

* 60 запросов в минуту
* до 1 000 000 запросов в месяц

При превышении лимита:

```json
{
  "cod": "429",
  "message": "Too Many Requests"
}
```

---

## Рекомендации

### Кэширование

Рекомендуется сохранять результаты запросов локально для снижения нагрузки на API.

### Обработка ошибок

* Проверяйте HTTP статус
* Обрабатывайте поле `cod`
* Реализуйте retry при временных ошибках (500, 502)

---

## Примечания

* API работает только по HTTPS
* Ответы всегда в JSON
* Временные метки — UNIX timestamp
* Для точных данных используйте координаты (lat/lon)

---

## Заключение

Weather API позволяет интегрировать погодные данные в любые приложения. Правильная работа с параметрами, ошибками и лимитами обеспечивает стабильную интеграцию сервиса.
